# 元の試験要件



## コーディング試験問題：AI搭載型ナレッジベース「TechInsight」の構築

### 1. 試験の概要

技術記事データを基盤としたナレッジマネジメントシステム「TechInsight」を構築する。
本試験では、基本的なデータ管理機能（CRUD）の実装に加え、AIを用いた「セマンティック検索」の統合能力、および実用的なシステム設計能力を評価する。

### 2. 技術要件

- **Backend:** Python / FastAPI
- **Frontend:** Next.js / TypeScript / React
- **Database:** PostgreSQL
- **AI/ML:** 任意のモデルを用いたベクトル化（Embedding）の実装
- **Infrastructure:** Docker Compose で全サービスがワンコマンドで起動すること

### 3. 実装タスク

#### タスク1：データマイグレーションと環境構築

- 提供された `articles.csv`（1,000行）を読み込み、データベースに保存するスクリプトまたはプロセスを作成する。
  - ※提供されたデータは 1,000 件だが**実際には 1 万件ほどに増えると仮定**してクエリ効率などを考えた上で設計・実装すること。
- Docker Compose を使用し、データベース、API、フロントエンドが連携した状態で起動するように構築する。
  - `docker compose up` でマイグレーションが行われ、Web アプリケーションが起動するまでを行う。

#### タスク2：Backend API の実装

- **CRUD API:** 記事の取得（検索含む）、新規作成、更新、削除を行うエンドポイント。
- **検索 API の詳細:** ユーザーが入力した自然言語のクエリに対して、記事本文との「**意味的な類似度**」に基づいた検索結果を返す機能。使用するモデルや類似度の計算手法、検索のロジックは各自の判断に任せる。

#### タスク3：Frontend の実装

- **記事一覧・検索画面:** 記事を一覧表示し、キーワードまたはセマンティック検索を実行できる UI。
- **記事詳細表示:** 記事の全内容を確認できるモーダル、あるいはそれに準ずる表示機能。
- **管理機能:** 記事の投稿、既存記事の編集および削除ができるインターフェース（モーダルやフォーム）。

### 4. 補足

- すべて評価者のパソコンで完全再現できるようにする。ローカルで全機能が正しく動作すること。
- API キーの使用は認めるが、**評価者は API キーを持たない**前提で再現できること。
- 一人での開発だが、**チーム開発を意識した設計・実装**を行う。
- **保守運用、スケーラビリティを考慮**する。
- コード品質、ディレクトリ構成、パフォーマンスへの配慮、UX の工夫など、プロフェッショナルなエンジニアとして重要だと考える要素を自由に盛り込む。
- その他にこのサービスをブラッシュアップさせるものがあれば、各自の判断で適宜追加する。

- 実装したリポジトリ（public）を共有。README を正しく書く。
- 簡易 DB 設計書、簡易 API 設計書を提出。
- Web アプリケーションの実装の説明、工夫した点について以下の観点で述べる：
  - UI/UX の観点
  - DB の観点
  - チーム開発を意識した観点
  - 保守運用、スケーラビリティの考慮の観点
  - その他の観点
